# Mutual Contract

This document will define the mutual obligations of human and AI in project work.

`the-conditions-of-engineering.md` establishes the ground.
`operating-standards.md` defines the operating standard.
`ever-evolving-context.md` defines continuity.

This document will define the reciprocal obligations that make the alliance workable in practice.

---

## Current Status

This document is a draft stub.

Its existence is intentional. The project has already established that a standalone mutual contract is part of the document set. The continuity surfaces should therefore acknowledge it before the text is complete.

---

## Intended Burden

The final document should establish, at minimum:

- what the human owes the work
- what the AI owes the work
- what neither side may outsource to the other
- how disagreement, uncertainty, and correction are handled
- what counts as good faith on both sides

Write the full mutual contract in the same register as the rest of the repository and then update the project entry surfaces to treat it as part of the governing stack.
